Open-source - PAXI Framework

All our experiments for blockchain consensus protocols are conducted on a general framework called PAXI. PAXI framework is an open-source project mainly developed in a distributed system lab at SUNY-BUFFALO. PAXI framework is a coordination framework for prototyping and evaluation of consensus and replication protocols. PAXI has a fixed set of replicas N hardcoded in the configuration file before the framework launched. PAXI's network communication type is point-to-point and provides most of the elements that any replication protocol needs including network communication, state machine of a key-value store, client API and multiple types of quorum systems.

PAXI allows consensus protocols to be fairly evaluated in a common environment. PAXI has falexible system envirnoment for better evaluation such as varying workloads, and deployment size. PAXI framework is a module based with many built-in variety of consensus protocols. The framework allows us to compare many consensus protocols such as PBFT, Tendermint and other Paxos protocols written in GoLang in the same environment under identical workloads.
The framework provides extended abstractions to be shared between all Paxos variants, including location-aware configuration, network communication, client library with RESTful API, and a quorum management module.
