A Summary of Timing in Distributed Systems

Before getting into the Practical BFT (PBFT), I should introduce timing models (message latency) and failure models (stopping or doing arbitrary behavior). Timing model has three types:
1- The synchronous model is an upper bound (T) on a message latency and an upper bound (P) on the relative difference in speed between nodes.
2- The asynchronous model removes both upper bounds (T) and (P). Messages can take arbitrarily long to reach peers and each node can take an arbitrary amount of time to respond.
3- The partially synchronous is a mix of the two: upper bounds exist for (T) and (P) but the protocol designer does not know them. Protocol can use tactics like timeout to show when a node should retry sending the message.
